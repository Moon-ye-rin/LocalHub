# 실행 가이드

## Windows 빠른 실행

프로젝트 루트에서 다음 파일을 순서대로 실행합니다.

1. `start_backend.bat`
2. `start_frontend.bat`

접속 주소:

- 프론트엔드: `http://localhost:5173`
- Swagger API 문서: `http://localhost:8000/docs`
- 상태 확인: `http://localhost:8000/health`

정상 상태에서는 Health 응답에 다음 값이 포함됩니다.

```json
{
  "locations": 12512,
  "visible_locations": 11598,
  "visible_by_region": {
    "서울": 7102,
    "경기": 4496
  }
}
```

## 백엔드 수동 실행

```bash
cd backend
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env
python -m app.scripts.init_db
uvicorn app.main:app --reload
```

## 프론트엔드 수동 실행

새 터미널에서 실행합니다.

```bash
cd frontend
npm install
copy .env.example .env
npm run dev
```

## 데이터 다시 적재

현재 DB의 지역정보를 서울·경기 JSON으로 전부 교체하려면 다음 명령을 실행합니다.

```bash
cd backend
python seed_locations.py app/data --replace
```

원본 12,512건을 저장하고, 화면/API에서는 `firstimage`가 있는 11,598건만 노출합니다. 게시글·댓글 데이터는 유지됩니다.

## OpenAI 연동

`backend/.env`에서 설정합니다.

```env
USE_OPENAI=true
OPENAI_API_KEY=발급받은_키
OPENAI_MODEL=gpt-5-mini
```

실제 키가 포함된 `.env`는 Git에 커밋하지 않습니다.


## 카카오톡 공유 설정

`frontend/.env`에 카카오 JavaScript 키를 입력합니다.

```env
VITE_PUBLIC_SITE_URL=http://localhost:5173
VITE_KAKAO_JAVASCRIPT_KEY=발급받은_JavaScript_키
```

배포 환경에서는 `backend/.env`의 `FRONTEND_PUBLIC_URL`도 실제 Netlify 주소로 변경합니다. 카카오디벨로퍼스에서 프론트엔드 도메인을 JavaScript SDK 도메인에, 백엔드 Render 도메인을 제품 링크 관리 웹 도메인에 등록해야 합니다.

## A* 길찾기

지역정보 상세에서 현재 장소를 출발지로 사용하고, 도착 장소를 검색하거나 가까운 장소를 선택한 뒤 `경로 찾기`를 누릅니다.

- 지도 타일: OpenStreetMap
- 도로망 조회: Overpass API
- 최단 경로 계산: 백엔드 A* 알고리즘
- 기본 지원 거리: 직선거리 45km 이내
- 외부 도로망 조회가 필요하므로 인터넷 연결이 필요합니다.

공용 Overpass 서버가 지연될 경우 잠시 후 다시 시도하거나 `backend/.env`의 `OVERPASS_URL`을 다른 호환 서버로 변경할 수 있습니다.
