# LocalHub 기능 추가 v1.6

## 1. 지역정보 전체 조회

- `/locations` 지역 선택에 `전체`, `서울`, `경기` 제공
- 전체 선택 시 API에는 지역 조건을 생략하여 서울·경기 데이터를 함께 조회
- `GET /api/locations?region=전체` 직접 요청도 지원
- 개발 구현 안내 문구는 사용자 화면에서 제거

## 2. 게시글 첨부 이미지 비율 유지

- 게시글 상세 이미지를 `object-fit: contain`, `width: auto`, `height: auto`로 변경
- 원본 비율을 보존하고 화면 너비 및 뷰포트 높이 안에서 자동 축소
- 업로드 미리보기도 contain 방식으로 표시

## 3. 지역정보 댓글 수정·삭제

댓글 등록 정보:

- 닉네임: 미입력 시 익명
- 댓글
- 별점 0~5점
- 수정·삭제용 비밀번호 4자 이상

API:

- `POST /api/locations/{contentid}/comments`
- `PUT /api/locations/{contentid}/comments/{comment_id}`
- `DELETE /api/locations/{contentid}/comments/{comment_id}`

수정·삭제 요청마다 저장된 비밀번호를 다시 확인합니다. 기존 v1.5 DB는 실행 시 `password`, `updated_at` 컬럼과 수정 시각 트리거를 자동 추가합니다.

## 4. 실시간 알림 및 접속자 현황

- WebSocket: `/ws/notifications?client_key={브라우저키}`
- 게시글 등록 시 연결된 사용자에게 `post_created` 이벤트 전송
- 알림 종 아이콘에서 최근 10건 확인 및 게시글 상세 이동
- `presence` 이벤트로 현재 접속자 수 표시
- 브라우저 localStorage의 client key를 사용해 동일 브라우저 여러 탭을 한 명으로 집계

현재 구현은 단일 서버 프로세스 기준입니다. 다중 worker 배포 시 Redis pub/sub 도입이 필요합니다.
