# LocalHub v1.8 기능 추가

## 1. 백엔드 의존성

- `backend/requirements.txt`에 `tzdata>=2025.2` 추가
- Windows 및 일부 최소 Linux 이미지에서도 `ZoneInfo("Asia/Seoul")` 사용 가능

## 2. 소셜 공유

게시글 상세와 지역정보 상세 화면에 다음 버튼을 추가했습니다.

- 카카오톡 공유
- 공유 링크 복사

카카오톡 공유는 Kakao JavaScript SDK의 `Kakao.Share.sendDefault()` 기본 피드 템플릿을 사용합니다.

### 프론트엔드 환경변수

`frontend/.env`:

```env
VITE_API_BASE_URL=http://localhost:8000
VITE_PUBLIC_SITE_URL=http://localhost:5173
VITE_KAKAO_JAVASCRIPT_KEY=카카오_JavaScript_키
```

카카오디벨로퍼스 앱 설정에서 다음 도메인을 등록해야 합니다.

- JavaScript SDK 도메인: 프론트엔드 도메인
- 제품 링크 관리 웹 도메인: 백엔드 공유 URL 도메인

로컬 테스트 시 사용하는 프론트엔드·백엔드 localhost 도메인과, 배포 시 Netlify·Render 도메인을 각각 등록합니다.

## 3. OG 메타데이터

- SPA 기본 OG/Twitter 메타데이터를 `frontend/index.html`에 구성
- 게시글·지역정보 상세 진입 시 브라우저 메타데이터를 콘텐츠별로 변경
- 소셜 크롤러가 JavaScript를 실행하지 않아도 미리보기를 읽을 수 있도록 백엔드 공유 랜딩 페이지 추가

공유 랜딩 URL:

```text
GET /share/posts/{post_id}
GET /share/locations/{contentid}
```

랜딩 페이지는 콘텐츠 제목, 설명, 대표 이미지, 링크를 OG/Twitter 태그로 제공한 후 실제 Vue 상세 페이지로 이동합니다.

## 4. 배포 환경변수

Render 백엔드:

```env
FRONTEND_PUBLIC_URL=https://YOUR_NETLIFY_SITE.netlify.app
```

Netlify 프론트엔드:

```env
VITE_API_BASE_URL=https://YOUR_RENDER_SERVICE.onrender.com
VITE_PUBLIC_SITE_URL=https://YOUR_NETLIFY_SITE.netlify.app
VITE_KAKAO_JAVASCRIPT_KEY=카카오_JavaScript_키
```

카카오 JavaScript 키는 브라우저 SDK 식별용 공개 키이지만, REST API 키·Admin 키와 혼동하거나 소스코드에 직접 하드코딩하지 않습니다.
